# 2026-06-16 — Project scaffolding

## What was done

Set up the **Explainers** repo: a no-build, static collection of interactive HTML
explainers (one self-contained `index.html` per topic), to be deployed on GitHub
Pages. Modeled on explainers.blog (Erik Kennedy) and Paras Chopra's explainers.

### Research
- Read both reference sites. Key findings: Paras' repo is **100% HTML** (CSS/JS
  inlined), each explainer a self-contained `index.html` in its own slug folder,
  with a root landing page of cards. Content philosophy (Erik Kennedy): "accurate
  AND accessible", "you don't understand it until you can predict it", concrete
  hook → numbered question-headings → intuition-before-math → "well ackshually 🧐"
  asides → prediction section → closure.

### Decisions (confirmed with user)
- Style guide lives in **`CLAUDE.md`** (intentionally overrides the global
  "keep CLAUDE.md empty" rule for this project).
- Explainers are **self-contained single files** (inline CSS+JS), matching Paras.
- Built **structure only** for now — no sample topic yet.

### Files created
- `CLAUDE.md` — the explainer-generation style guide (main deliverable).
- `template/index.html` — canonical skeleton: design tokens, reading layout, TOC,
  `setupCanvas()` demo harness (HiDPI + reduced-motion safe), example demo,
  "ackshually" `<details>`, prediction box, footer, commented KaTeX include.
- `index.html` — landing page rendering cards from an inline `EXPLAINERS`
  manifest; empty-state shown until the first explainer exists.
- `README.md`, `ARCHITECTURE.md`, `LICENSE` (MIT/digster), `.gitignore`,
  `.nojekyll`, `PROMPT.md`.
- `package.json`, `playwright.config.js`, `tests/explainers.spec.js` — Playwright
  smoke tests that auto-discover every explainer folder.

## How to continue
- To add an explainer: copy `template/` → `<slug>/`, fill it in per `CLAUDE.md`,
  prepend a manifest entry in `index.html`, run `npm test`.
- Deploy: GitHub Pages → Deploy from branch → `main` / root.

## Fractals explainer (first real explainer)

Authored `fractals/index.html` via the pipeline and registered it on the landing
page (prepended to `EXPLAINERS`). Arc: romanesco hook → self-similarity → Koch
iteration → coastline paradox → fractal dimension (D = log N / log s, hand-rolled
HTML, no KaTeX) → Mandelbrot (z²+c) → applications → Sierpiński prediction →
synthesis. Six+ demos, all reusing the template's `setupCanvas`/`bindSlider`
harness; the Mandelbrot explorer is a standalone per-pixel `ImageData` renderer
(640px internal, click-to-zoom, iteration slider, presets) rather than the rAF
loop. Koch geometry (`kochStep`/`kochCurve`/`kochSnowflake`/`fitToCanvas`) is
shared by the Koch, coastline, and dimension demos.
